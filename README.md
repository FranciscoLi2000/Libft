# Libft

### Libft 项目的学习目标解析

#### 1. **核心目标：构建个人基础工具库**
- **重新实现标准库函数**（如 `ft_strlen`、`ft_memcpy`）：
深入理解底层实现（如指针操作、内存布局），避免依赖现成库，为后续项目（如 `printf`、`get_next_line`）奠定基础。
- **实现扩展功能**（如 `ft_split`、`ft_itoa`）：
掌握动态内存管理（`malloc`/`free`）、字符串处理技巧，培养防御式编程思维（处理 `NULL`、越界访问）。

#### 2. **关键技能培养**
- **内存管理**：
避免内存泄漏（如 `ft_split` 需逐层释放）、理解堆栈差异（如静态函数 `static` 的作用）。

- **代码规范**：
通过 Norminette 强制统一风格（缩进、函数长度、命名），适应工业级代码标准。
- **Makefile 编写**：
掌握自动化构建（`all`、`clean`）、理解编译选项（`-Wall -Werror`）的重要性。

#### 3. **链表操作的进阶训练（Bonus 部分）**
- **数据结构基础**：实现链表增删查改（如 `ft_lstadd_back`、`ft_lstclear`），为后续算法项目（如排序、图遍历）打基础。
- **泛型编程**：通过 `void *content` 支持任意数据类型，理解抽象设计的价值。

#### 4. **工程思维提升**
- **模块化设计**：分离头文件（`libft.h`）与实现，提升代码复用性。
- **测试驱动开发**：编写自定义测试用例（如边界值测试），减少依赖 Moulinette 反馈。

---

### 给正式学生的学习建议：
1. **优先级管理**：
- 先完成**基础函数**（Part 1），再处理**附加功能**（Part 2），最后尝试 Bonus。
- 使用 `valgrind` 检测内存泄漏，避免在简单错误上浪费时间。

2. **利用社区资源**：
- 参与 Slack/Intranet 讨论，复用他人已验证的工具函数（如快速 `ft_itoa` 实现）。
- 在 Defense 中主动解释设计思路（如为何选择递归而非迭代），展现深度思考。

3. **长期维护 Libft**：
- 后续项目中不断优化（如添加哈希表、动态数组），将其作为个人技术能力的“名片”。
- 例如：在 `minishell` 项目中使用 `ft_split` 解析命令，在 `cub3D` 中使用 `ft_putstr_fd` 输出日志。

---

### 总结：
从 Piscine 到正式学生，是从“生存模式”到“创造模式”的转变。Libft 不仅是首个项目，更是未来所有 C 项目的基石——**代码质量决定天花板，工程习惯决定效率**。保持对细节的执着（如处理 `malloc` 失败），同时拥抱更大挑战（如实现一个内存安全的 `ft_strjoin`），你将逐步从“码农”蜕变为“工程师”。
